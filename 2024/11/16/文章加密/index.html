Hexo 文章加密

发表于 2024-11-16

四、对TOC进行加密（可选）

如果你的文章包含目录（TOC），并且你希望也对目录进行加密，你可能需要修改Next主题的模板文件。具体步骤如下：

  1. 找到并修改模板文件：

    • 在Next主题的layout文件夹中找到与文章相关的模板文件（如article.ejs）。
    • 根据插件的文档或示例代码，修改模板文件中的TOC生成代码，以便在加密时隐藏目录。

  2. 重新生成和预览博客：

    • 运行Hexo的生成和预览命令，查看修改后的效果。

五、注意事项

  • 安全性：虽然这种方法可以为你的文章提供基本的加密保护，但请注意，它并不是一种非常安全的加密方式。如果你的博客源码是公开的（如存放在GitHub上），别有用心之人仍然有可能通过查看源码找到加密密码。因此，对于非常敏感的内容，建议使用更安全的加密和保护方法。
  • 兼容性：确保你使用的Hexo版本和Next主题版本与加密插件兼容。如果不兼容，可能会出现错误或无法正常工作的情况。
  • 备份：在修改配置文件或模板文件之前，建议备份原始文件，以便在出现问题时能够恢复。

这是个啥

  • 首先, 这是 Hexo 生态圈中 最好的 博客加密插件

  • 你可能需要写一些私密的博客, 通过密码验证的方式让人不能随意浏览.

  • 这在 wordpress, emlog 或是其他博客系统中都很容易实现, 然而 hexo 除外. :(

  • 为了解决这个问题, 让我们有请 “hexo-blog-encrypt”.

特性

  • 一旦你输入了正确的密码, 它将会被存储在本地浏览器的 localStorage中. 按个按钮, 密码将会被清空. 若博客中有脚本, 它将会被正确地执行.

  • 支持按标签加密.

  • 所有的核心功能都是由原生的 API 所提供的. 在 Node.js中, 我们使用 Crypto. 在浏览器中, 我们使用 Web Crypto API.

  • PBKDF2, SHA256 被用于分发密钥, AES256-CBC 被用于加解密, 我们还使用 HMAC 来验证密文的来源, 并确保其未被篡改.

  • 我们广泛地使用 Promise 来进行异步操作, 以此确保线程不被阻塞.

  • 加密页面多主题支持, 现在已经支持的主题有 [default, xray], 更多的主题正在开发中.

  • 过时的浏览器将不能正常显示, 因此, 请升级您的浏览器.
